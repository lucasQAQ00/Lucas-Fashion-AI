# 04 ERP利润分析系统

## 项目目标

整合女装业务的销售、采购、库存、退款及平台费用数据，形成可信的单款和多维利润分析结果。

## 使用场景

- 计算商品、订单、渠道、店铺和时间周期的毛利与净利。
- 识别高销量低利润、退货侵蚀利润和库存积压问题。
- 支持选款、定价、补货和促销复盘。

## 技术方案

- 使用 pandas 或 Polars 进行数据清洗、关联和指标计算。
- 首期通过脱敏 CSV/Excel 导入，稳定后再接入 ERP 授权接口或只读数据库。
- 建立统一指标口径和数据校验规则，计算过程保留明细以便审计。
- 后续可使用 Streamlit 或 BI 工具呈现分析看板。

## 后续开发步骤

1. 明确 ERP 数据字段、成本组成和利润计算口径。
2. 准备脱敏样例并实现多来源数据校验。
3. 完成订单级利润计算和异常数据报告。
4. 增加商品、渠道和时间维度聚合分析。
5. 构建看板、权限控制、定时更新和结果对账。
